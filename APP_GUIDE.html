Shorten Link

Hướng dẫn chạy demo và dùng thư viện rút gọn link

App này gồm một thư viện .NET có thể đóng gói thành NuGet, một demo ASP.NET Core API, và một React UI để tạo, xem, copy, redirect và deactivate short link.

Chạy nhanh Dùng API Dùng giao diện Nhúng vào app .NET Cấu hình Kiểm release

App này dùng để làm gì?

Tạo short link

Nhập URL dài, tùy chọn expiry, app sẽ tự sinh short code random để copy và chia sẻ.

Redirect

Truy cập /{code} để chuyển tới original URL. Link bị deactivate sẽ trả trạng thái gone.

Thư viện reusable

Project khác có thể dùng package ShortenLink.AspNetCore để gọi AddShortenLink(...) và map endpoints.

1. Chạy nhanh trên máy local

Chạy các lệnh bên dưới từ thư mục root của repo: D:\8.Codex\shorten-link.

Kiểm tra build và test

dotnet build ShortenLink.slnx
dotnet test ShortenLink.slnx

Chạy demo API

dotnet run --project src\ShortenLink.Api\ShortenLink.Api.csproj --launch-profile https

Sau khi API chạy, mở Swagger tại:

https://localhost:7154/swagger

API cũng mở cổng HTTP http://localhost:5188, cổng này được Vite frontend proxy tới trong lúc dev.

Chạy React UI

cd .\src\ShortenLink.Web
npm install
npm run dev

Mở giao diện tại:

http://localhost:5173
Luồng dùng bình thường: chạy API trước, chạy Web sau, mở http://localhost:5173, nhập URL dài, tạo short link, copy short URL và thử truy cập short URL đó.

2. Dùng API trực tiếp

Endpoint Mục đích
POST /api/short-links Tạo short link mới.
GET /api/short-links/{code} Xem chi tiết short link theo code.
GET /{code} Redirect tới original URL.
DELETE /api/short-links/{code} Deactivate short link.
GET /api/health Kiểm tra API còn sống.

Ví dụ tạo link bằng PowerShell

$body = @{
  originalUrl = "https://example.com"
} | ConvertTo-Json

Invoke-RestMethod `
  -Method Post `
  -Uri "https://localhost:7154/api/short-links" `
  -ContentType "application/json" `
  -Body $body

Thử redirect

https://localhost:7154/{code-tra-ve-tu-api}

3. Dùng giao diện React

  1. Mở http://localhost:5173.
  2. Nhập URL gốc vào form tạo link.
  3. Chọn expiry nếu muốn.
  4. Bấm tạo link để app tự sinh short code random.
  5. Copy short URL được trả về và mở trên trình duyệt.
  6. Vào trang detail bằng code được tạo để xem thông tin hoặc deactivate link.

Nếu truy cập code không tồn tại, demo sẽ đưa về fallback page của frontend trong môi trường development.

4. Nhúng thư viện vào app .NET khác

Với app ASP.NET Core, package chính nên dùng là ShortenLink.AspNetCore.

Cách dùng bằng project reference khi dev local

dotnet add reference ..\shorten-link\src\ShortenLink.AspNetCore\ShortenLink.AspNetCore.csproj

Cách dùng bằng local NuGet package

New-Item -ItemType Directory -Force .\.nupkg
dotnet pack ..\shorten-link\ShortenLink.slnx -c Release -o .\.nupkg
dotnet nuget add source .\.nupkg --name shorten-link-local
dotnet add package ShortenLink.AspNetCore --source .\.nupkg

Program.cs tối thiểu

using ShortenLink.AspNetCore;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddShortenLink(builder.Configuration);

var app = builder.Build();

app.UseShortenLinkRateLimiting();
app.MapShortenLinkEndpoints();

app.Run();

Package nào để làm gì? ShortenLink.AspNetCore là entry point cho host ASP.NET Core. ShortenLink.Core chứa contracts và domain behavior. ShortenLink.Infrastructure chứa EF Core persistence cho SQLite/PostgreSQL.

6. Kiểm package và release readiness

Trước khi đưa package cho app khác dùng hoặc chuẩn bị publish, chạy bộ kiểm sau:

dotnet build ShortenLink.slnx --verbosity minimal
dotnet test ShortenLink.slnx --verbosity minimal
dotnet pack ShortenLink.slnx -c Release --verbosity minimal
.\scripts\release-dry-run.ps1
.\scripts\smoke-consumer-package.ps1

release-dry-run.ps1 kiểm package artifacts nhưng không publish. Nếu truyền -Publish, script sẽ fail để tránh publish nhầm.

smoke-consumer-package.ps1 tạo một app ASP.NET Core sạch trong .tmp, install ShortenLink.AspNetCore, chạy SQLite default mode và kiểm create/detail/redirect/delete.

Giữ artifact để xem thủ công

.\scripts\release-dry-run.ps1 -KeepArtifacts
.\scripts\smoke-consumer-package.ps1 -KeepArtifacts

7. Lỗi thường gặp

Triệu chứng Cách xử lý
Frontend gọi API không được Chạy API bằng launch profile https trước. Vite proxy mặc định tới http://localhost:5188.
Không mở được Swagger Kiểm tra API còn chạy và mở https://localhost:7154/swagger.
NuGet restore bị lỗi mạng Các smoke/release command có thể cần truy cập api.nuget.org để restore dependency.
Docker/PostgreSQL/Redis không có Không sao cho local default. Để UsePostgres=false, Cache.Enabled=false, Analytics.Enabled=false.
